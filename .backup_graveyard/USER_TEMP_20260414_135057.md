# 2. LibProjectM v4.x.x Core & Framebuffer Architecture

### The Render-Record-Display Problem (Bottleneck Warning)
ProjectM needs to render frames, the UI needs to display them inside a QML layout, and the recorder needs to capture them simultaneously without tanking the UI framerate. **Do not run ProjectM on the Qt Main UI Thread.**

### Optimal Architecture: Asynchronous Shared FBO + PBO
Create a dedicated `ProjectMRenderContext` running on its own QThread.
1.  **Rendering:** Force Qt 6 to use the OpenGL RHI (or Vulkan if ProjectM v4 supports it natively on your target machine, but OpenGL is safest for ProjectM).
2.  **The FBO:** The render thread draws ProjectM output into an Offscreen Framebuffer Object (`QOpenGLFramebufferObject`).
3.  **To QML (Simultaneous Viewing):** Create a custom `QQuickFramebufferObject` or `QQuickRhiItem`. Use a shared OpenGL context to pass the FBO texture ID to the QML Scenegraph. The QML thread simply renders a textured quad, costing virtually 0 CPU/GPU.
4.  **To Video (Simultaneous Recording):** When recording is toggled, use **Pixel Buffer Objects (PBOs)** with double-buffering. 
    *   *Why?* Calling `glReadPixels` directly to encode video will stall the GPU pipeline, halving your framerate. 
    *   *How:* Asynchronously map the PBO to system memory (`glMapBufferRange`), pass the raw RGB data to your FFmpeg encoding thread via a lock-free queue (using `std::atomic` or `moodycamel::ConcurrentQueue`), and let FFmpeg encode the mp4/mkv in the background.

# 5. Potential Performance Bottlenecks to Note & Fix

1.  **QML Text Overlays over OpenGL:** Heavy SVG rendering or complex text animations inside the QML Scenegraph can cause frame-pacing stutters. 
    *   *Fix:* Cache SVG items using `layer.enabled: true` and `layer.smooth: true` in QML to render them to textures once rather than rasterizing every frame.
2.  **Suno API Thread Blocking:**
    *   *Fix:* Use `QNetworkAccessManager` asynchronously or `std::async` + libcurl if doing raw C++. Never block the GUI thread.
3.  **ProjectM v4 Preset Swapping Stutter:** Loading complex `.milk` shaders involves parsing and compiling GLSL. 
    *   *Fix:* ProjectM v4 supports background loading. Ensure you use the thread-safe API to pre-load the *next* preset in the playlist before the transition occurs.
4.  **String Copies for ID3/Lyrics:**
    *   *Fix:* Use C++23 `std::string_view` for parsing the ID3 tags and lyrics database lines to avoid massive allocation overhead during rapid playback updates.
